# packages/db — Database Layer

## Purpose

Drizzle ORM configuration, schema definitions, and database client for Neon PostgreSQL.

## Structure

```
src/
├── schema/           # Drizzle table definitions
│   └── index.ts      # Re-export all schemas
├── migrations/       # Generated by drizzle-kit
└── index.ts          # Export db client + schema
```

## Conventions

### Schema Definitions

- One file per domain in `schema/` (e.g., `users.ts`, `accounts.ts`)
- Re-export everything from `schema/index.ts`
- Use `numeric(15, 2)` for monetary amounts — never float
- Use `uuid` for primary keys
- Always add `createdAt` and `updatedAt` timestamps

### Database Client

- Uses `@neondatabase/serverless` driver with Drizzle's `neon-http` adapter
- Connection string from `DATABASE_URL` environment variable
- Exported as `db` from package root

### Migrations

- Generated with `drizzle-kit generate`
- Applied with `drizzle-kit migrate`
- Never edit migration files manually

### Rules

- **Never** import from `apps/`
- Can import from `@rumbo/shared` for type alignment
- Keep queries in app services, not in this package
